# 最適化：設計指針

同期コストを抑え、安定した体験を作るための指針です。

## 1. 同期頻度を減らす

高頻度な同期は帯域制限により詰まりやすくなります。

**例**: 残り時間を毎秒同期するのではなく、
- 「タイマー終了時刻」を同期する
- ローカルの現在時刻との差分で表示する

## 2. 状態同期とイベント同期の使い分け

- **後から入室するプレイヤーにも状態再現が必要** → 同期変数
- **同時再生だけ必要で後からの再現不要** → `SendCustomNetworkEvent`

## 3. Continuous を使うべき場面

`Continuous` は位置補間など連続更新が必要な場合に限定します。
それ以外は `Manual` で同期頻度を制御する方が安全です。
